Conditions in OSWF are similar to functions except that they return a value that equates to boolean true or false.

Each condition must be defined as a child of the conditions element. This element has one attribute called type, which can either be AND or OR. When using the type AND all the condition elements must evaluate to true or the overall condition will return as false. When using the type OR only one condition element must evaluate to true for the overall condition to pass. If you require more complex conditional logic, then consider implementing it yourself using the Condition interface. Note that the type can be omitted if the conditions element contains only one condition.

It is possible to nest conditions by simply specifying additional <conditions> children elements under a <conditions> element, which enables you to express more complex logical operations than a simple AND or OR on a set of conditions on the same level.

Below is a list of some of the builtin Conditions

HasTypedMapValue; (hasProperty)

  • name (argument) - The name of a TypedMap to assign a value to
  • value (argument) - The String value to be tested

Test a TypedMap value for String equality. If the value is not a String the 'toString' method will be invoke prior to the 'equals'. If the 'name' is not found in the TypedMap an IllegalArgumentException exception will be thrown

HasRole; (hasRole)

  • actor (transient) - The actor's role to tested
  • role (argument) - The actor's role to tested

Gets the current actor from the transient variables map then uses the SecurityManager to test if this actor has this role; return true or false

HasStatusOf; (hasStatusOf)

  • stepId (argument) - The stepId to be tested
  • status (argument) - The status of the stepId

Returns true or false if the stepId has this status.

IsStepOwner; (isStepOwner)

  • actor (transient) - The current actor
  • stepId (argument) - The stepId to be tested

Returns true or false if the stepId is owned by the actor.

IsStringPropertyEqual; (isPropertyEqual)

  • name (argument) - The name of a TypedMap to test
  • value (argument) - The String value to be tested against

Returns true or false if the value of the TypedMap property reference by name is equal. The 'toString' method is applied to the value before the String 'equals' methods is used to test.
