Functions - Builtin

OSWf comes with several very built-in utility functions, all implementing the interface net.sf.oswf.FunctionProvider. For more detailed information, please see the javadocs for these utility functions. Below is only a brief description of each utility function. All classes are found in the net.sf.oswf.functions package.

AssertEquals; (assertEquals)

Takes two String arguments expected and actual and compares them using the String method equals. Throws a WorkflowException if the evaluation returns false. Used primarily for testing, not advised for production release code.

BeanShellFunctionProvider

Type='Beanshell' used to interprept bsh code in an XML Workflow Description.

ClearStep; (clearStep)

  • stepId (argument) - The step number(s) in the Current Step table; Multiple steps are comma delimited
  • status (argument) - History Step status, may be null

Remove a step from the Current Step table and optionally move it to the History Step table and assigns an exit status. Calling with a StepId which is not in the Current Steps table is safe as nothing will happen.

GetStringProperty; (getString)

  • name (argument) - The name of a name/value pair in the TypedMap
  • variable (argument) - The name of the variable to assign in the transientVars map

A very useful function to move TypedMap values into transient values.

SetActor; (setActor)

  • name (argument) - The name of actor

Sets the actor from within the Workflow Description. Useful for automated or system events.

SetDateProperty; (now)

  • name (argument) - The name of a TypedMap to set to the current datetime

Creates or sets a TypedMap name/value pair to the current datetime.

SetStringProperty; (setString)

  • name (argument) - The name of a TypedMap to assign a value to
  • value (argument) - The value to be assignged
  • default (argument) - Use this value if the 'value' argument is null

Creates or sets a TypedMap name/value pair to the current datetime.

TimerTask; (timer)

  • action (argument) - The actionId to fire on the current step
  • duration (argument) - An amount of time to wait before firing

Fire and action in the current step of the current process instance. If the actionId is not appropriate for this step nothing will occur. Note that this is implemented as a Java TimerTask in the JVM which will disappear if the JVM is restarted.
